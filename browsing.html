How Browsers Render a Web Page

Step-by-step journey from HTML & CSS to pixels on the screen.

1. HTML File

Browser receives the HTML document from the server. This is the raw markup containing elements like headings, paragraphs, images, buttons, etc.

↓

2. Parsing HTML

The browser reads the HTML character by character and starts understanding its structure.

↓

3. Tokenization

HTML is broken into tokens such as opening tags, closing tags, attributes, and text content.

↓

4. DOM Tree

Browser converts the HTML into a tree-like structure called the Document Object Model (DOM). JavaScript can access and modify this tree.

⇣

5. CSS File

Browser downloads and reads all CSS files and style rules.

↓

6. CSSOM Tree

CSS rules are converted into the CSS Object Model (CSSOM), which stores all styling information.

↓

7. DOM + CSSOM

Browser combines structure (DOM) and styling (CSSOM) together.

↓

8. Render Tree

A render tree is created containing only visible elements along with their computed styles.

↓

9. Layout (Reflow)

Browser calculates the exact size and position of every visible element.

↓

10. Paint

Colors, borders, shadows, text and images are painted onto different layers.

↓

11. Composite

Layers are merged together and displayed on the user's screen. DONE
